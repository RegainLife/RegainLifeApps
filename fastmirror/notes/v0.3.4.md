# FastMirror 0.3.4 (2026-03-04)

## 概要
- トレイ格納中に同期イベントを受信した際、同期アニメが表示されない不具合を修正しました。
- 同一内容ファイルの連続投入時に同期先でリネーム逸脱復旧が誤発火する問題を修正しました。
- 常駐同期とトレイ常駐時のアニメ連動を安定化し、回帰テストを再実施しました。

## 変更内容
- `WatchEvent::Renamed` で `internal_event_ignores` を評価し、自アプリ起因のリネームイベントを復旧対象から除外。
- `try_cleanup_duplicate_destination_files()` で source 側に実在する対応ファイルを保持し、孤立重複のみ整理対象へ限定。
- トレイアニメ制御で `requested` と `timer running` 状態を分離し、`WM_START_ANIM` で確実に `SetTimer` を開始。
- worker/SyncFinished 側の先行 stop を廃止し、UI keepalive 経路へ停止判定を一本化。
- `test_minimize_message_tray_watch_on.ps1` と `diag_large_file_sync.ps1` を release ビルドで再検証（SYNC/CPUともに通過）。
